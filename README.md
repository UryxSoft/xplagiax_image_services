## What is xplagiax Images Search Services?

**XplagiaX ISS** is a production-grade microservice that combines three AI capabilities into a single REST API:

- **AI vs Human Detection** — classifies whether an image was generated by AI or created by a human, using a fine-tuned SigLIP model (2024)
- **Visual Similarity Search** — finds visually similar images using CLIP embeddings and approximate nearest-neighbor search via Qdrant
- **Plagiarism Analysis** — detects copied, modified, cropped, recoloured, or watermarked versions of images with configurable similarity thresholds
- **Patent & Reverse Image Search** — searches the web and patent databases using external APIs with intelligent rotation and circuit breaking

Built to run at scale: async ML inference, distributed object storage via SeaweedFS, vector indexing via Qdrant, and full observability out of the box.

### Why this stack?

| Component | Choice | Reason |
|-----------|--------|--------|
| **Embeddings** | CLIP ViT-B/32 | Best open-source visual similarity model |
| **AI Detection** | SigLIP fine-tuned | State-of-the-art AI vs Human classifier (2024) |
| **Vector DB** | Qdrant + HNSW | Sub-100ms ANN search, payload indexes, cosine similarity |
| **Object Storage** | SeaweedFS | Purpose-built for billions of small files, no metadata DB needed |
| **Cache / Queue** | Redis | Atomic counters, embedding cache, async job queue |
| **API Server** | Gunicorn + gevent | Non-blocking I/O, single worker keeps one model copy in RAM |

## Features

- **Idempotent indexing** — same image always produces the same UUID (SHA-256 based). Re-uploading never creates duplicates.
- **Async ML inference** — Flask responds in `<100ms`. Heavy CLIP/SigLIP inference runs in a dedicated RQ worker process.
- **Graceful degradation** — Redis down? Service keeps running. SigLIP failed to load? CLIP search still works. Every failure is isolated.
- **Zero vector poisoning protection** — corrupt images are rejected before touching Qdrant. No silent bad data.
- **Path traversal prevention** — all filenames and group IDs are sanitised with allowlists before filesystem or storage operations.
- **Structured JSON logging** — every log line includes `request_id`, `endpoint`, `group_id`, and `duration_ms` for full traceability.
- **Prometheus metrics** — 20+ metrics including P95 latency, model confidence distribution, cache hit rates, and collection size.
- **SeaweedFS storage** — images stored in distributed object storage. Switching from local to SeaweedFS requires only one env var change.

## Similarity Thresholds

| Score | Match Type | Interpretation |
|-------|-----------|----------------|
| ≥ 0.97 | `EXACT_COPY` | Same image, possibly recompressed or resized |
| 0.90 – 0.97 | `MODIFIED_COPY` | Edited: crop, color change, watermark, filter |
| 0.80 – 0.90 | `HIGH_SIMILARITY` | Visually related, possible derivative work |
| 0.70 – 0.80 | `MODERATE_SIMILARITY` | Some visual relation — manual review recommended |
| < 0.70 | No match | Unrelated images |

## Quick Start

### Prerequisites

- Docker 24+
- 8 GB RAM minimum (12 GB recommended)
- 20 GB free disk space

### 1 — Clone the repository

```bash
git clone https://github.com/youruser/xplagiax.git
cd xplagiax
```

### 2 — Create the Docker network

```bash
docker network create xplagiax-net
```

### 3 — Start infrastructure services

```bash
# Redis
docker run -d --name redis-server --network xplagiax-net \
  -p 6379:6379 -v redis_data:/data --restart unless-stopped \
  redis:7.2-alpine redis-server --maxmemory 2gb --maxmemory-policy allkeys-lru

# Qdrant
docker run -d --name qdrant --network xplagiax-net \
  -p 6333:6333 -p 6334:6334 -v qdrant_data:/qdrant/storage --restart unless-stopped \
  qdrant/qdrant:v1.10.1

# SeaweedFS Master
docker run -d --name seaweedfs-master --network xplagiax-net \
  -p 9333:9333 -v seaweedfs_data:/data --restart unless-stopped \
  chrislusf/seaweedfs:latest master \
  -ip=seaweedfs-master -ip.bind=0.0.0.0 -port=9333 -mdir=/data

# SeaweedFS Volume  (wait 5 seconds after master)
docker run -d --name seaweedfs-volume --network xplagiax-net \
  -p 8080:8080 -v seaweedfs_data:/data --restart unless-stopped \
  chrislusf/seaweedfs:latest volume \
  -mserver=seaweedfs-master:9333 -ip=seaweedfs-volume \
  -ip.bind=0.0.0.0 -port=8080 -dir=/data -max=0

# SeaweedFS Filer  (wait 5 seconds after volume)
docker run -d --name seaweedfs-filer --network xplagiax-net \
  -p 8888:8888 -v seaweedfs_filer_data:/data --restart unless-stopped \
  chrislusf/seaweedfs:latest filer \
  -master=seaweedfs-master:9333 -ip=seaweedfs-filer \
  -ip.bind=0.0.0.0 -port=8888
```

```env

## Security

- API key authentication via `X-API-Key` or `Authorization: Bearer`
- Rate limiting: 30 req/min, 500 req/hour per IP (configurable)
- Hard image size limit: 20 MB (enforced at Nginx + WSGI + validation layers)
- Magic byte validation — file type verified by content, not extension
- Path traversal protection on all filesystem and storage operations
- Deterministic UUIDs from SHA-256 content hash — prevents enumeration
- Internal paths and stack traces never exposed in API responses
- Confirmation token required for destructive admin operations

## Características

- **Indexación idempotente** — la misma imagen siempre produce el mismo UUID (basado en SHA-256). Re-subir nunca crea duplicados.
- **Inferencia ML asíncrona** — Flask responde en `<100ms`. El pesado proceso de inferencia CLIP/SigLIP corre en un worker RQ dedicado.
- **Degradación elegante** — ¿Redis caído? El servicio sigue funcionando. ¿SigLIP falló? La búsqueda CLIP sigue operando. Cada fallo está aislado.
- **Protección contra vector cero** — las imágenes corruptas son rechazadas antes de tocar Qdrant. Sin datos silenciosamente malos.
- **Prevención de path traversal** — todos los nombres de archivo y group IDs son sanitizados con allowlists.
- **Logs JSON estructurados** — cada línea incluye `request_id`, `endpoint`, `group_id` y `duration_ms` para trazabilidad completa.
- **Métricas Prometheus** — más de 20 métricas incluyendo latencia P95, distribución de confianza del modelo y tasas de cache hit.
- **Almacenamiento SeaweedFS** — imágenes en almacenamiento objeto distribuido. Cambiar de local a SeaweedFS: un solo cambio de variable de entorno.

docker network create xplagiax-net
```

### 3 — Iniciar servicios de infraestructura

```bash
# Redis
docker run -d --name redis-server --network xplagiax-net \
  -p 6379:6379 -v redis_data:/data --restart unless-stopped \
  redis:7.2-alpine redis-server --maxmemory 2gb --maxmemory-policy allkeys-lru

# Qdrant
docker run -d --name qdrant --network xplagiax-net \
  -p 6333:6333 -p 6334:6334 -v qdrant_data:/qdrant/storage --restart unless-stopped \
  qdrant/qdrant:v1.10.1

# SeaweedFS Master
docker run -d --name seaweedfs-master --network xplagiax-net \
  -p 9333:9333 -v seaweedfs_data:/data --restart unless-stopped \
  chrislusf/seaweedfs:latest master \
  -ip=seaweedfs-master -ip.bind=0.0.0.0 -port=9333 -mdir=/data

# SeaweedFS Volume  (esperar 5 segundos después del master)
docker run -d --name seaweedfs-volume --network xplagiax-net \
  -p 8080:8080 -v seaweedfs_data:/data --restart unless-stopped \
  chrislusf/seaweedfs:latest volume \
  -mserver=seaweedfs-master:9333 -ip=seaweedfs-volume \
  -ip.bind=0.0.0.0 -port=8080 -dir=/data -max=0

# SeaweedFS Filer  (esperar 5 segundos después del volume)
docker run -d --name seaweedfs-filer --network xplagiax-net \
  -p 8888:8888 -v seaweedfs_filer_data:/data --restart unless-stopped \
  chrislusf/seaweedfs:latest filer \
  -master=seaweedfs-master:9333 -ip=seaweedfs-filer \
  -ip.bind=0.0.0.0 -port=8888
```

## Seguridad

- Autenticación por API key via `X-API-Key` o `Authorization: Bearer`
- Rate limiting: 30 req/min, 500 req/hora por IP (configurable)
- Límite duro de 20 MB por imagen (Nginx + WSGI + validación)
- Validación por magic bytes — el tipo de archivo se verifica por contenido
- Protección contra path traversal en todas las operaciones de storage
- UUIDs deterministas desde SHA-256 — previene enumeración
- Rutas internas y stack traces nunca expuestos en respuestas de API
- Token de confirmación requerido para operaciones admin destructivas
